# OpenClaw 文档管理章程

> 📜 **文档管理的"宪法"** - 所有文档都必须遵守此章程
>
> **版本**: 1.0
> **生效日期**: 2026-03-07
> **维护者**: 全体团队成员

---

## 📜 第一章 总则

### 第一条 目的

本章程旨在建立统一的文档管理规范，确保：
1. 所有文档都有合适的存放位置
2. AI 和人类都能快速找到所需文档
3. 避免重复内容和信息分散
4. 便于知识沉淀和传承

### 第二条 适用范围

本章程适用于：
- 所有 OpenClaw 项目相关文档
- 团队成员编写的技术文档
- AI 自动生成的各类文档
- 会议记录、报告、方案等

### 第三条 基本原则

1. **位置优先**: 写文档前先确定位置
2. **模板先行**: 使用统一模板编写
3. **命名规范**: 遵循统一的命名规则
4. **关联引用**: 相关文档互相引用
5. **定期整理**: 定期清理和归档

## 📁 第二章 文档分类与位置

### 第四条 文档类型定义

| 类型代码 | 文档类型 | 存放位置 | 命名规则 |
|---------|---------|---------|---------|
| DR | 日报 (Daily Report) | `06-日报/个人/` | `YYYY-MM-DD-日报.md` |
| WR | 周报 (Weekly Report) | `06-日报/周报/` | `YYYY-Www-周报.md` |
| MR | 月报 (Monthly Report) | `06-日报/月报/` | `YYYY-MM-月报.md` |
| LL | 教训 (Lesson Learned) | `05-运维/踩坑教训/` | `YYYY-MM-DD-{主题} 踩坑记录.md` |
| PR | 方案 (Proposal) | `02-项目/{项目}/方案设计/` | `{主题}方案.md` |
| RP | 报告 (Report) | `02-项目/{项目}/检查报告/` | `YYYY-MM-DD-{主题}报告.md` |
| RF | 复盘 (Retrospective) | `07-复盘/` | `YYYY-MM-DD-{主题}复盘.md` |
| SK | 技能 (Skill) | `skills/{skill-name}/` | `SKILL.md` |
| API | API 密钥 | `09-私人/` | **统一管理，禁止分散** |

### 第五条 项目目录结构

```
02-项目/
├── {项目名}/
│   ├── 方案设计/          # 所有方案、计划
│   ├── 检查报告/          # 所有检查、验收报告
│   ├── 实施记录/          # 执行过程记录
│   ├── 会议纪要/          # 项目会议记录
│   └── {其他}/           # 根据项目需要创建
```

### 第六条 禁止行为

1. ❌ 禁止在文档中硬编码 API Key
2. ❌ 禁止将私人文档提交到公开仓库
3. ❌ 禁止在项目目录外创建项目文档
4. ❌ 禁止使用无意义的文件名

## 📝 第三章 文档编写规范

### 第七条 文档结构

所有文档必须包含：
1. **Front Matter** (元数据)
2. **标题** (清晰明确)
3. **正文** (结构清晰)
4. **相关文档** (关联引用)
5. **位置标注** (脚注形式)

### 第八条 Front Matter 规范

```yaml
---
type: {文档类型}
tags: [标签 1, 标签 2]
created: {创建日期}
updated: {更新日期}
author: {作者}
status: {状态}
---
```

### 第九条 位置标注规范

所有文档必须在开头和结尾添加位置标注：

```markdown
<!--
📂 文档位置：路径/到/文档.md
📋 模板类型：模板名称
🤖 AI 指引：简要说明
-->

*📍 位置：`路径/到/文档.md` | 模板：`模板名称.md`*
```

## 🤖 第四章 AI 协作规范

### 第十条 AI 写文档流程

1. **确定类型**: 询问"这是什么类型的文档？"
2. **查找位置**: 根据类型确定存放位置
3. **选择模板**: 使用对应模板创建
4. **填写内容**: 按模板结构填写
5. **添加关联**: 链接相关文档
6. **标注位置**: 添加位置标注

### 第十一条 AI 上传文档规范

AI 在上传/保存文档时必须：
1. 检查文档位置是否正确
2. 检查文件名是否符合规范
3. 检查是否已有相同主题文档
4. 添加适当的标签和关联

### 第十二条 文档位置查询

AI 应该能够快速回答：
- "XXX 文档应该放哪里？"
- "有没有写过类似文档？"
- "这个文档的模板是什么？"

## 🔐 第五章 安全管理

### 第十三条 敏感信息管理

1. **API Key**: 统一存放在 `09-私人/API 密钥管理总表.md`
2. **密码**: 使用钥匙串管理，文档中只引用
3. **Token**: 同 API Key 管理方式

### 第十四条 文档权限

| 目录 | 访问权限 | 说明 |
|------|---------|------|
| `09-私人/` | 仅本人 | 私人文档和密钥 |
| `00-私人/` | 仅本人 | 私人笔记 |
| 其他目录 | 团队成员 | 工作相关文档 |

## 📊 第六章 质量检查

### 第十五条 检查项目

每次创建文档时检查：
- [ ] 文档位置是否正确
- [ ] 文件名是否符合规范
- [ ] 是否使用了模板
- [ ] 是否添加了 Front Matter
- [ ] 是否添加了位置标注
- [ ] 是否关联了相关文档
- [ ] 是否避免了重复内容

### 第十六条 定期检查

| 检查类型 | 频率 | 检查内容 |
|---------|------|---------|
| 位置检查 | 每周 | 文档是否放在正确位置 |
| 重复检查 | 每周 | 是否有重复内容 |
| 密钥检查 | 每月 | 是否有硬编码密钥 |
| 归档检查 | 每月 | 是否需要归档旧文档 |

## 🛠️ 第七章 工具使用

### 第十七条 文档工具

| 工具 | 用途 | 位置 |
|------|------|------|
| Templates | Obsidian 模板 | `.obsidian/templates/` |
| doc-locator.py | 文档定位脚本 | `shared/scripts/` |
| today-report.py | 日报/周报生成 | `shared/scripts/` |

### 第十八条 快捷命令

```bash
# 定位文档位置
python3 shared/scripts/doc-locator.py locate 日报

# 检查重复
python3 shared/scripts/doc-locator.py check "主题"

# 创建文档
python3 shared/scripts/doc-locator.py create 日报

# 生成日报
python3 shared/scripts/today-report.py daily
```
